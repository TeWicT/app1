# Изменения с коммита `069a91d1de2e641324ffd38666c873551ac291fe` (включительно) до текущего состояния

## Кратко
- Реализованы страницы **обсуждений работ** (чат/форум) для студентов и преподавателей с **реальным временем через WebSocket (Django Channels)**, добавлена поддержка **Redis channel layer** для продакшена.
- Добавлен профиль **Администратора** и страница **«Админ‑панель»** для управления темами всех преподавателей (UI по образцу страницы преподавателя).
- Доработаны ограничения и UX вокруг тем: **скрыть/показать тему** с запретом скрытия при наличии заявок/принятых студентов и визуальными индикаторами скрытых тем.
- Исправлены проблемы продакшен‑интеграции WebSocket/CSRF: trusted origins, маршрутизация `/ws/...`, Nginx upgrade headers, диагностика ASGI.
- Доработана страница **«Поиск и просмотр работ»**: вывод групп за все годы, сортировка годов по убыванию, корректное отображение кафедры «Не указана».
- Синхронизация legacy‑данных: `index.clj` перезаписывается при изменениях темы/регистрации/файлов, улучшен импорт legacy‑архивов (в т.ч. `:department nil`).
- Исправлена подсветка «просрочено» на странице поиска работ: теперь определяется **по `index.clj :creation-time`** и заданным дедлайнам; пустые ячейки не подсвечиваются, добавлены tooltip‑подсказки.

## Технологии и библиотеки
- Django Channels: WebSocket, `AsyncWebsocketConsumer`, ASGI‑роутинг.
- Redis: `channels-redis` как backend channel layer (по env‑флагу).
- Nginx: проксирование WebSocket (Upgrade/Connection headers).

## Модели (`webd_core/models.py`)
- Добавлен `AdminProfile`:
  - Доступ к странице «Админ‑панель» определяется наличием записи профиля.
- Добавлены модели обсуждений:
  - `DiscussionThread` (обсуждение для темы)
  - `DiscussionMessage` (сообщения)
- Доработки `Topic`:
  - Флаг видимости темы для студентов: `is_active` (скрыть/показать).
  - Разрешено пустое значение кафедры у преподавателя/тем (используется «Не указана» в UI).

## Middleware (`webd_core/middleware.py`)
- В `request` добавляется `admin_profile` (аналогично `student_profile`/`teacher_profile`), чтобы шаблоны и views могли корректно проверять роль.

## Представления (`webd_core/views.py`)
### Обсуждения
- `student_discussion`: доступ студенту только при **утверждённой заявке** на тему.
- `teacher_discussions`: список тем преподавателя с участниками и переходом в обсуждение.
- `discussion_view`: единая страница обсуждения (GET/POST) + проверка доступа.

### Админ‑панель
- `admin_panel_view`: управление темами всех преподавателей (создание/редактирование/удаление), UI максимально повторяет «Предложить тему».

### Видимость темы (скрыть/показать)
- `_handle_topic_toggle_active`:
  - запрет скрытия темы, если есть **pending/approved** заявки;
  - визуальная маркировка скрытых тем.

### «Поиск и просмотр работ»
- Сбор групп/лет для фильтров расширен: отображаются **все группы за все годы**, годы отсортированы по убыванию, «Все» внизу.
- Исправлено отображение кафедры: если пусто — показывается **«Не указана»**.
- Подсветка просрочки документов:
  - проверка делается **по `index.clj :creation-time`**, а не по `Document.uploaded_at`;
  - дедлайны от календарного года начала работы (year+1):
    - для **Пр. отчет / Пр. ЭП**: после **01.01** следующего года — просрочено;
    - для остальных документов: после **01.06** следующего года — просрочено.
  - добавлены tooltip‑подсказки с дедлайном и датой загрузки.

### Синхронизация legacy `index.clj`
- `index.clj` перезаписывается:
  - при изменении данных в «Регистрация работы»;
  - при загрузке/удалении файлов;
  - при изменениях/подтверждениях/удалениях тем (в т.ч. очищение Enrollment при удалении темы).

## Известные нюансы и поведение
- Подсветка просрочки намеренно считается **по `index.clj`**, а не по времени импорта/записи в БД.
- Если `index.clj` отсутствует или не содержит `creation-time` для конкретного ключа файла, ячейка **не считается просроченной** (чтобы не красить ложноположительно).
